# Scripts

## Generate synthetic utility outage report vCons (primary dataset)

The script **`generate_utility_outage_vcons.py`** produces **synthetic** utility outage report conversations in vCon 0.0.1 format. The scenario: **AI virtual assistant** handles intake (no human dispatcher), verifies the caller’s **address via caller ID**, and provides **real-time technician ETA** (and optional SMS/app updates).

> **Disclaimer:** All data generated by this script is **synthetic**. It is not from real utilities, callers, or locations. Use only for demos, MCP tooling, and vCon reference.

### Report types

- **Gas leak**: Caller reports gas smell; AI verifies address via caller ID, creates urgent ticket, gives technician ETA and 5-minute-out SMS.
- **Power outage**: Caller reports power out; AI verifies address via caller ID, creates outage ticket, gives restoration or technician ETA and app/SMS updates.

### Usage

From the repo root:

```bash
# Generate 42 vCons into day folders 18–24 (main dataset)
python3 scripts/generate_utility_outage_vcons.py --out . --count 42 --seed 42

# Default: 8 vCons into 18–24
python3 scripts/generate_utility_outage_vcons.py --out .

# Custom count
python3 scripts/generate_utility_outage_vcons.py --out . --count 20

# Custom start date and base URL for audio
python3 scripts/generate_utility_outage_vcons.py --out . --start-date 2025-06-01 --base-url https://example.com/audio
```

### Options

| Option | Default | Description |
|--------|---------|-------------|
| `--out`, `-o` | `.` | Output directory (vCons written under day subdirs if `--day-folders` used) |
| `--day-folders` | `18 19 20 21 22 23 24` | Spread vCons into these subdirs (use `none` or override) |
| `--count`, `-n` | (all scenarios) | Number of vCons to generate |
| `--base-url` | (empty) | Base URL for `dialog[].url` |
| `--start-date` | `2025-05-18` | Start date for call timestamps (YYYY-MM-DD) |
| `--seed` | (none) | Random seed for reproducibility |

### Viewing

The viewer (`streamlit run vcon_viewer.py`) loads vCons from **911_calls/** and any numbered day folders (`18/`–`24/`). Run the utility outage generator with `--out .` and default `--day-folders` to populate day folders.

### Generating audible speech (TTS) for vCons

To get **real speech** (so you hear the transcript when you press play in the viewer), use TTS:

```bash
pip install gtts
python3 scripts/generate_tts_audio_for_vcons.py --root .
```

This reads the transcript from each vCon and creates an `.mp3` with spoken content (Google TTS). Refresh the viewer to hear the audio. Scans **911_calls** and numbered dirs by default; use `--dirs 911_calls` to limit.

### Generating silent placeholder audio (.mp3 or .wav)

The viewer can play a file per vCon if `uuid.mp3` or `uuid.wav` exists. To create **silent** placeholders (duration only, no speech):

```bash
python3 scripts/generate_mp3_for_vcons.py --root .
# Or .wav only: python3 scripts/generate_mp3_for_vcons.py --root . --wav-only
```

With **ffmpeg** installed the script produces `.mp3`; otherwise `.wav`. Use `--force` to overwrite existing files.

### Creating your own gas leak examples

Add new gas leak (or power outage) scenarios by appending to the `SCENARIOS` list in **`generate_utility_outage_vcons.py`**. Each scenario is a dict with:

| Field | Description |
|-------|-------------|
| `report_type` | `"gas_leak"` or `"power_outage"` |
| `caller_name` | Caller’s full name |
| `caller_tel` | E.164-style phone (e.g. `"+15559876001"`) |
| `caller_mailto` | Caller email |
| `address` | Address used for caller ID verification and ticket (must match what the AI will say) |
| `transcript` | Full conversation: **alternating lines**, **AI first**, then Caller. Include: (1) AI greeting, (2) caller describes issue, (3) AI verifies address from caller ID, (4) caller confirms, (5) AI creates ticket + safety advice + technician ETA + SMS/app mention, (6) brief close. |
| `summary` | One or two sentences: what was reported, address, and outcome (ticket + ETA). |

**Transcript pattern (gas leak):**

1. AI: “Thank you for calling City Gas & Power. I'm your virtual assistant. How can I help you today?”
2. Caller: Describe where/how (kitchen, meter, hallway, furnace, neighbor’s house, etc.).
3. AI: “I have your address as [ADDRESS] from your caller ID. Is that correct?” (use the same `address` as in the scenario).
4. Caller: Confirm or correct.
5. AI: “I've created an urgent gas report for [ADDRESS]. [Safety advice]. A technician is on the way. ETA is X minutes. You'll get a text when they're 5 minutes away.”
6. Caller: Short reply.
7. AI: “You're welcome. Stay safe.”

**Example minimal gas leak scenario** (copy and edit in `generate_utility_outage_vcons.py`):

```python
{
    "report_type": "gas_leak",
    "caller_name": "Your Name",
    "caller_tel": "+15559876999",
    "caller_mailto": "you@email.com",
    "address": "123 Main St",
    "transcript": """Thank you for calling City Gas & Power. I'm your virtual assistant. How can I help you today?

I smell gas in the [location].

I have your address as 123 Main St from your caller ID. Is that correct?

Yes.

I've created an urgent gas report for 123 Main St. [Safety tip]. A technician is on the way. ETA is 25 minutes. You'll get a text when they're 5 minutes away.

Thank you.

You're welcome. Stay safe.""",
    "summary": "Caller reports gas smell at [location] at 123 Main St. AI verifies address via caller ID, creates urgent ticket, technician ETA 25 minutes.",
},
```

Built-in gas leak examples in the script include: building/hallway, backyard meter, kitchen (stove off), basement water heater, strong hallway smell (evacuation), forklift strike at warehouse, neighbor’s address, furnace, and bathroom/water heater.

## Generate synthetic 911 call vCons (optional)

The script **`generate_911_vcons.py`** produces **synthetic** 911 emergency call conversations in vCon 0.0.1 format (dispatcher + caller), for separate demos or MCP testing.

> **Disclaimer:** All 911 call data generated by this script is **synthetic**. It is not from real 911 systems, dispatchers, or callers. Do not use for any purpose that assumes real emergencies, locations, or identities.

### Scenarios

- Medical, fire, burglary, car accident, domestic disturbance, suspicious person, roadside emergency

### Usage

```bash
# 8 vCons into 911_calls/
python3 scripts/generate_911_vcons.py

# Custom output and count
python3 scripts/generate_911_vcons.py --out my_911_data --count 20
```

See above (utility outage) for how to view data in the main viewer (e.g. copy into a numbered day folder) or use with your MCP server.

## Demo: show this data consumed by the MCP server

**`demo_mcp_consume.py`** – Ingest this repo’s vCons into [vcon-mcp](https://github.com/vcon-dev/vcon-mcp) via its REST API, then list and get one vCon to show the data being consumed by the MCP server. Use this for your hackathon demo.

**Prerequisites:** vcon-mcp running in **HTTP mode** (see vcon-mcp: `MCP_TRANSPORT=http`, `MCP_HTTP_PORT=3000`), Supabase configured.

```bash
pip install requests
export MCP_BASE_URL=http://localhost:3000
export API_KEY=your-key   # only if vcon-mcp has API_KEYS set
python3 scripts/demo_mcp_consume.py
```

Options: `--dir 911_calls` (default), `--base-url`, `--api-key`, `--dry-run`. See main [README](../README.md) section “Using vCon MCP with this repo”.
